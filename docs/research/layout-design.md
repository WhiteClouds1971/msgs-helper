# 面杀辅助工具 — 布局设计方案

**版本**：2.2 | **日期**：2026-06-18 | **状态**：范围已锁定

---

## 1. 设计理念

放弃传统 Web 工具站的"首页 → 子页"层级导航，转向 **iPhone 原生应用式的交互模型**：

- 每个工具 = 一个独立的全屏页面，占满整个浏览器视口（100vw × 100vh）
- 没有首页仪表盘，用户在工具之间直接切换，不经过中间页
- 所有操作收敛于一枚悬浮球——长按切换工具、短按唤出辅助功能、双击返回
- 横竖屏均可全屏展示，不受 480px 单一宽度限制

**设计隐喻**：悬浮球 = **悬浮印章**。一枚始终浮于所有界面之上的朱砂印章，长按揭开"奏章叠层"（工具快照），短按展开"辅佐之扇"（搜索/语音/设置/关于）。

```
┌────────────────────────────────────────────────────┐
│              iPhone 风格交互架构                      │
├────────────────────────────────────────────────────┤
│                                                    │
│  工具层：全屏页面（100vw × 100vh）                    │
│  ┌──────────────────────────────────────────────┐  │
│  │                                              │  │
│  │     工具 A · 全屏内容                          │  │
│  │     （竖屏 / 横屏 自适应）                      │  │
│  │                                              │  │
│  │                                        ●     │  │ ← 悬浮球
│  └──────────────────────────────────────────────┘  │
│                                                    │
│  长按 ≥500ms ────────→  卡片堆叠切换器（AppSwitcher） │
│  ┌──────────────────────────────────────────────┐  │
│  │  ┌──┐ ┌──┐ ┌──┐  ← 工具快照卡片层叠            │  │
│  │  │ B│ │ C│ │ D│     滑动选择·点击切换           │  │
│  │  └┬─┘ └──┘ └──┘     (按最近使用时间降序排列)     │  │
│  │   └─── 当前工具 A（最前层，金色描边）            │  │
```

## 2. 架构总览

| 层级 | 元素 | z-index | 说明 |
|------|------|---------|------|
| 工具层 | 全屏 `<router-view />` | 0-200 | 单个工具占满视口，横竖屏自适应 |
| 卡片堆叠遮罩 | AppSwitcher 遮罩 | 550 (`--z-app-switcher`) | 全屏暗色遮罩，覆盖工具内容 |
| 卡片堆叠容器 | AppSwitcher 卡片 | 551 | 3D 层叠卡片，滑动选择工具 |
| 扇形菜单遮罩 | FanMenu 透明遮罩 | 570 | 拦截菜单外点击，关闭菜单 |
| 扇形菜单项 | FanMenu 按钮 | 575 | 从悬浮球位置扇形展开 |
| 悬浮层 | FloatingBall | 600 (`--z-tooltip`) | 始终在最上。Modal 打开时降至 399 |

> **z-index 动态调整**：当 Modal（`--z-modal: 400`）打开时，悬浮球 z-index 降至 399（低于 Modal），关闭后恢复 600。防止在 Modal 中误触悬浮球。

**路由策略**：每个工具一个路由 `/tool/:name`，通过 `tools.map()` 动态生成路由配置。Vue Router 管理前进/后退。浏览器前进/后退按钮与 Pinia 历史栈自动同步（见第 4 节）。

## 3. 交互详细设计

### 3.1 悬浮球（FloatingBall）

始终悬浮于所有界面之上，可拖拽，松手后吸附屏幕边缘。

```
┌──────────────────────────────┐
│                              │
│                              │
│     [工具全屏内容]            │
│                              │
│                        ┌──┐ │
│                        │印│ │  ← 朱砂红圆形，印章意象
│                        └──┘ │     直径 56px
└──────────────────────────────┘
```

**设计规范**：

| 属性 | 值 | 说明 |
|------|-----|------|
| 尺寸 | 56×56px 圆形 | 大于 44px 触控最小值，适合桌面操作距离 |
| 颜色 | `--accent-red` 朱砂红底 + 白色"面"字 | `.seal-stamp` 印章效果，微旋转 -2° |
| 默认位置 | 右下角，距边缘 16px | `right: var(--space-4); bottom: var(--space-4)` |
| 层级 | `z-index: var(--z-tooltip)` (600) | Modal 打开时动态降至 399 |
| 阴影 | `--shadow-lg` | 悬浮感 |
| 吸附动画 | `transition: transform 300ms var(--ease-enter)` | **仅用 transform，不触发 layout**（符合 DESIGN_SYSTEM.md 反模式约束） |
| 拖拽实现 | PointerEvents（`pointerdown/move/up`） | fallback: TouchEvents。全局 `touch-action: manipulation` 已消除 300ms 延迟 |
| 拖拽时防滚动 | 悬浮球元素 CSS `touch-action: none` | 阻止拖拽时页面滚动。不影响其余区域滚动 |
| 事件绑定 | `pointerdown` 在悬浮球上，`pointermove/up` 在 `document` 上 | 手指滑出球体后仍能跟踪 |
| 位置持久化 | 会话期内 Pinia 内存存储，刷新重置为默认右下角 | 后期对接数据模型统一管理持久化（`localStorage` 方案待定） |

#### 3.1.1 手势状态机 🔴 关键规格

悬浮球承载 4 种手势。以下状态机精确定义每种手势的判定条件和互斥规则：

```
TOUCHDOWN (pointerdown)
  │
  ├─ 记录起始坐标 (startX, startY)
  ├─ 启动长按计时器 (500ms)
  │
  ├─ [移动距离 > 10px 且长按计时器未触发]
  │    → 取消长按计时器
  │    → 进入 DRAGGING 状态
  │    → pointermove 时实时更新 translate
  │    → pointerup 时吸附最近水平边缘
  │
  ├─ [长按计时器到达 500ms 且移动距离 ≤ 10px]
  │    → 进入 LONG_PRESS 状态
  │    → 打开 AppSwitcher（卡片堆叠切换器）
  │    → 悬浮球在此之前不触发任何视觉变化
  │
  └─ [pointerup 且持续 < 300ms 且移动距离 ≤ 10px 且长按计时器未触发]
       → 进入 SHORT_PRESS 状态
       → 启动双击等待窗口 (300ms)
       │
       ├─ [300ms 内检测到第二次 pointerdown]
       │    → 取消 SHORT_PRESS
       │    → 进入 DOUBLE_TAP 状态
       │    → 返回上一个工具（历史栈 pop）
       │
       └─ [300ms 超时无第二次点击]
            → 打开 FanMenu（扇形菜单）
```

**关键参数**：

| 参数 | 值 | 说明 |
|------|-----|------|
| 长按阈值 | 500ms | 达到此时间触发 AppSwitcher |
| 移动取消阈值 | 10px | 手指移动超过此距离取消长按，转入拖拽 |
| 短按最大时长 | 300ms | touchend 在此时间内为短按 |
| 双击窗口 | 300ms | 短按后等待第二次点击的时间窗口 |
| 吸附方向 | 仅水平吸附（左或右边缘） | 垂直位置保持松手时的 Y 坐标 |
| 吸附判定 | 悬浮球中心距左右边缘 ≤ 屏幕宽 50% 则吸附该侧 | 正中间时（50% 恰好相等）→ 吸附右侧（默认侧） |
| 拖拽边界 | `translateX/Y` 限制在视口内 | 悬浮球不超出屏幕边界 |

**短按延迟说明**：双击检测使短按扇形菜单存在 ~300ms 延迟。这是有意为之的取舍——双击返回是高频操作，"扇形菜单慢 300ms"的代价低于"双击被误识别两次短按"。如需零延迟短按，可将返回功能移到扇形菜单第一项，取消双击手势。

#### 3.1.2 新手引导

悬浮球是无文字标签的红色圆形，新用户需要引导。策略：

1. **首次启动**：悬浮球旁显示 tooltip 气泡"长按切换工具"，3 秒后自动消失
2. **前 3 次使用**：每次短按扇形菜单打开时，菜单项附带文字标签（如"搜索""设置"）。第 4 次起仅显示图标
3. **引导记录**：`localStorage` 存储 `onboarding: { tooltipShown: true, fanMenuWithLabels: 3 }`

### 3.2 卡片堆叠切换器（长按）

长按悬浮球 500ms（且手指未移动超过 10px）后进入。全屏暗色遮罩 + 3D 层叠卡片。**这是工具切换的唯一入口**。

```
┌──────────────────────────────┐
│  · · · · · · · · · · · · · ·│  ← --bg-overlay 暗色遮罩 (z-index: 550)
│                              │
│    ┌──────────────────┐      │
│    │   工具 C (后层)    │      │  ← 缩小的快照卡片
│    └──────────────────┘      │     rotateY(-4°) translateZ(-80px) scale(0.94)
│      ┌──────────────────┐    │
│      │   工具 B (中层)    │    │  ← rotateY(-2°) translateZ(-40px) scale(0.97)
│      └──────────────────┘    │
│  ┌──────────────────────┐    │
│  │                      │    │
│  │   工具 A · 当前工具   │    │  ← 最前层，正常比例，金色描边高亮
│  │       (前层)         │    │     rotateY(0) translateZ(0) scale(1)
│  │                      │    │
│  └──────────────────────┘    │
│                              │
│  ← 左右滑动选择 · 上滑关闭   │  ← 底部提示文字
├──────────────────────────────┤
│        ✕ 关闭               │  ← 底部关闭按钮
└──────────────────────────────┘
```

**卡片排序**：按最近使用时间（LRU）降序排列。当前使用的工具始终在最前层。即：用户最近用过的工具紧跟在当前工具之后。

**3D 层叠参数**：

| 参数 | 值 | 说明 |
|------|-----|------|
| 容器 | `perspective: 1000px` | 3D 透视基准 |
| 每层 Y 旋转 | `max(-6°, -2° × index)` | 后层微微侧转，最多偏转 6° |
| 每层 Z 位移 | `-40px × index` | 后层推向深处 |
| 每层缩放 | `max(0.91, 1 - 0.03 × index)` | 后层缩小，最小 0.91 |
| 显示层数 | 最多 4 层可见 | 超过 4 层的卡片在后方不可见 |
| 可见卡片 DOM | 最多渲染 6 个（4 可见 + 前后各 1 缓冲） | 性能保护 |
| 当前卡片 | `rotateY(0) translateZ(0) scale(1)` + `.card--accent` | 金色描边高亮 |

**卡片快照内容**（Phase 1 静态卡片）：

每张卡片显示：
- 工具图标 / cover 缩略图
- 工具名称（`--font-display`，`--text-xl`）
- 主题色小条（体力卡朱砂红、势力卡古铜金、武将工具铜绿等）

**技术路线决策** 🔴：Phase 1 静态卡片定为**长期方案**，不再升级到 Keep-Alive 实时快照。理由：
- 在 3D `perspective` 容器中渲染真实 Keep-Alive 组件存在架构障碍（缓存实例属于 router-view 上下文，无法在 AppSwitcher 中二次挂载）
- 6 个完整工具组件在 3D 透视中同时渲染，移动端 GPU 纹理开销不可控
- 静态卡片已满足"快速识别工具"的核心需求，实时快照的实际价值有限

**交互**：

| 手势 | 行为 |
|------|------|
| 左右滑动 | 浏览卡片堆叠。滑动超过卡片宽度 30% 时吸附到相邻卡片；未达阈值弹回原位 |
| 点击前层卡片 | 切换到对应工具，关闭切换器 |
| 点击后层卡片 | 该卡片滑到最前层（不切换，继续浏览） |
| 点击遮罩空白区 | 关闭切换器，留在当前工具 |
| 点击 ✕ 按钮 | 关闭切换器 |
| 上滑手势 | 关闭切换器（快捷手势） |
| 快速连续滑动 | 动画进行中忽略新输入（`isAnimating` flag 锁定 ~300ms） |

**与 FanMenu 的互斥**：卡片切换器是全屏模态。打开时悬浮球隐藏（`opacity: 0`），关闭后恢复。FanMenu 不能在切换器之上打开。

**工具数极端情况**：

| 工具数 | 行为 |
|--------|------|
| 0 | 不触发切换器（系统至少有一个工具） |
| 1 | 不触发切换器（无其他工具可切换），长按无效 |
| 2-6 | 正常层叠显示，所有卡片可见 |
| 7-50 | 仅显示最近 6 张卡片（4 可见 + 2 缓冲）。切换器顶部显示搜索框，输入即过滤卡片 |

**性能优化**：
- 卡片元素在切换器打开时添加 `will-change: transform`，关闭后移除
- 仅渲染当前可见卡片 + 前后各 1 张缓冲区的 DOM（虚拟化策略）

### 3.3 扇形菜单（短按）

短按悬浮球（<300ms，且 300ms 内无第二次点击）展开。从悬浮球位置（右下角）向左上方扇形散射 4 个菜单项。

```
               🔍 搜索
          🎙 语音
     ⚙ 设置
ℹ️ 关于
 ╲
  ✦  ← 悬浮球（旋转 45°，朱砂→金色过渡）
```

**设计规范**：

| 属性 | 值 | 说明 |
|------|-----|------|
| 展开方向 | 左上方扇形（半径 130px） | 从右下角悬浮位置发出 |
| 菜单项数量 | 4 个 | 搜索、语音、设置、关于。主题切换合并到设置 Modal |
| 菜单项尺寸 | 44×44px 圆形 | 满足 44px 触控，避免 48px 时的相邻触控重叠 |
| 角度分布 | 搜索 25° / 语音 50° / 设置 75° / 关于 95° | 覆盖 70° 弧，弧长间距 ~57px（> 44px，无触控重叠） |
| 半径自适应 | `min(130px, 35vh)` | 极小高度屏幕不溢出 |
| 菜单项颜色 | `--bg-surface` 底 + `--accent-gold` 描边 | 与悬浮球朱砂红形成对比 |
| 入场动画 | 逐个弹出（stagger 50ms），总时长 ~250ms | `--ease-ink` 水墨扩散曲线 |
| 出场动画 | 统一反向收起（stagger 30ms），总时长 ~150ms | `--ease-out` 快出 |
| 动画中断 | Web Animations API（`element.animate()`） | 支持随时 cancel 并启动反向动画。不使用 CSS transition 以避免中断时状态不一致 |
| 菜单项标签 | 前 3 次使用附带文字标签，之后仅图标 | 新手引导的一部分（见 3.1.2） |
| 关闭方式 | 点击菜单外透明遮罩（z-index: 570） | 遮罩拦截点击事件，不传递到下层工具页 |
| 遮罩 | 覆盖全屏的半透明遮罩（`--bg-overlay` 极低透明度） | 视觉效果极淡，主要作用是事件拦截 + 视觉暗示"外面可点击" |

**菜单项功能详情**：

> ⚠️ **本期范围**：仅实现扇形菜单的展开/关闭交互和视觉骨架。每个子菜单项（搜索、语音、设置、关于）的具体内部功能**不在本期范围内**，后期单独开发。本期点击菜单项仅触发一个占位事件（console.log + 可选 toast），方便后续接入。

| 功能 | 图标 | 本期行为 | 后期规划 |
|------|------|---------|---------|
| 搜索 | 🔍 | 点击关闭扇形菜单，emit `@select('search')` | 全屏搜索 Modal |
| 语音 | 🎙 | 点击关闭扇形菜单，emit `@select('voice')` | TTS 朗读 + 播放控制 |
| 设置 | ⚙ | 点击关闭扇形菜单，emit `@select('settings')` | 设置 Modal（音效/数据/主题） |
| 关于 | ℹ️ | 点击关闭扇形菜单，emit `@select('about')` | 关于 Modal |

**与 AppSwitcher 的互斥**：扇形菜单打开期间，长按悬浮球 → 先关闭扇形菜单，再打开卡片切换器（`200ms` 过渡）。

**扇形菜单打开期间悬浮球行为**：
- 悬浮球旋转 45°
- 再次短按悬浮球 → 关闭菜单（与点击遮罩等效）
- 拖拽 → 关闭菜单 + 移动悬浮球
- 长按 → 关闭菜单 + 打开卡片切换器

### 3.4 全屏工具页

每个工具是一个独立的全屏页面，100vw × 100vh。悬浮球不占用布局空间。

```
竖屏 (portrait)：                  横屏 (landscape)：
┌──────────────┐          ┌──────────────────────┐
│ ← 体力卡  🎙 │          │ ← 体力卡         🎙  │
├──────────────┤          ├──────────────────────┤
│              │          │                      │
│  [工具内容]  │          │    [工具内容]         │
│              │          │                      │
│              │          │                 ●    │ ← 悬浮球
│              │          │                      │
│         ●    │          └──────────────────────┘
└──────────────┘
```

**页面结构**（竖屏）：

```
┌──────────────────────────────┐
│ ← 体力卡              🎙    │  ← 顶部栏（可选，简单工具可省略）
├──────────────────────────────┤
│                              │
│    [工具专属内容区域]         │  ← flex: 1
│    · 每个工具自主决定内部布局  │     **约束：必须使用 Design Token
│    · 支持竖屏/横屏自适应      │      （颜色/间距/字体通过 var(--*) 引用）
│                              │     触控目标 ≥ 44px，横竖屏自适应**
│                              │
└──────────────────────────────┘
```

**横竖屏自适应策略**：

| 场景 | 策略 |
|------|------|
| 计数器类工具（体力卡） | 竖屏纵向堆叠、横屏横向并排（如双人/多人时） |
| 记录类工具（花色/点数） | 竖屏单列时间线、横屏双列卡片网格 |
| 武将技能面板 | 竖屏整页滚动、横屏左右分栏（技能列表 + 详情） |
| 简单工具（大旗/标记） | 完全无顶部栏，内容居中全屏。仅靠悬浮球导航 |

**顶部栏**（可选）：

| 元素 | 说明 |
|------|------|
| 返回按钮 (←) | 可选。悬浮球双击也能返回上一个工具 |
| 工具名称 | `--font-display`，`--text-lg` |
| 语音入口 (🎙) | 可选。悬浮球扇形菜单已提供语音入口 |

> **DESIGN_SYSTEM.md 480px 约束说明**：100vw 全屏 + 横竖屏自适应是本布局的核心特性。`DESIGN_SYSTEM.md` 的 `max-width: 480px` 约束在移动端竖屏场景下仍然有效（设备宽度 ≤480px），桌面端调试时通过居中容器限宽。详见待确认事项 #9。

### 3.5 双击返回

双击悬浮球 → 返回上一个使用的工具（历史栈的 `pop`）。

- 历史栈仅剩当前工具（无法继续 pop）→ 双击无效（无操作）
- 首次启动（历史栈为空）→ 双击无效
- 扇形菜单打开中双击 → 关闭菜单 + 返回上一个工具

### 3.6 首次启动行为 🔴

新用户首次打开应用时没有历史栈、没有"上一个工具"。策略：

1. **默认工具**：进入工具注册列表的第一个工具（当前为体力卡）
2. **新手引导**：悬浮球旁显示 tooltip 气泡"长按切换工具"（3 秒消失）
3. **扇形菜单带标签**：前 3 次展开时菜单项附带文字标签

> 不自动弹出卡片切换器作为"隐式首页"——这会让新用户困惑（"为什么我什么都没做就弹出一个全屏东西"）。让用户先看到一个可用的工具，再通过悬浮球引导发现其余功能。

### 3.7 横屏下的适配

横屏时悬浮球移到屏幕右下角（使用相同 CSS 定位逻辑）。扇形菜单从悬浮球当前位置展开，角度分布不变。

**悬浮球横屏位置决策**：选择横屏右下角而非右侧中部的理由——保持"右下角"的肌肉记忆一致性；用户旋转设备后悬浮球仍在预期位置。如果后续用户测试反馈横屏时右下角偏远，可增加"常用位置"选项让用户自选。

**卡片堆叠在横屏下**：仍保持 3D 深度层叠（非水平排列）。v2.0 3.6 节示意有误——横屏不改变层叠方向。

### 3.8 加载页（Splash Screen，之后实现）

在 `index.html` 中以**内联 CSS + HTML** 实现，不依赖任何外部 JS/CSS 资源。

```html
<!-- index.html -->
<div id="app-splash">
  <div class="splash-seal">面杀</div>   <!-- 印章旋转动画 -->
  <div class="splash-ink-bar"></div>    <!-- 水墨横线扩散 -->
</div>
```

| 元素 | 动画 | 时长 |
|------|------|------|
| 印章 | 360° 持续旋转 + 呼吸缩放 | 循环 |
| 水墨线 | scaleX(0→1) + opacity 淡入 | 800ms 循环 |
| 背景 | 纯 `--bg` 底色 | 静态 |

## 4. 状态管理

| Store | 职责 | 技术 |
|-------|------|------|
| `toolNavigation` | 切换历史栈、当前工具路由、导航同步 | Pinia |
| `floatingBall` | 悬浮球位置（会话期内 Pinia 内存存储）、吸附边、当前状态（idle/dragging/fan-open/switcher-open） | Pinia |
| 各工具 store | 高频/跨组件工具用 Pinia；简单工具用组件本地状态 | 按需 |

**Pinia 历史栈与浏览器历史的同步** 🔴：

Pinia `toolNavigation` 栈**不作为独立真实来源**。它通过 `watch(() => route.path)` 从 Vue Router 的变化中**派生**：

- 路由变化 → `watch` 回调 → 栈 push / pop / replace
- 浏览器前进/后退按钮 → 路由变化 → 栈自动同步
- 双击悬浮球 → `router.back()` → 路由变化 → 栈自动 pop
- 卡片切换器选择工具 → `router.push()` → 路由变化 → 栈自动 push

这消除了文档 v2.0 中"两个独立历史来源不同步"的数据一致性问题。

**悬浮球位置持久化**（后期）：
- 本期：拖拽松手后位置保存在 Pinia `floatingBall` store 中，仅会话期内有效。刷新重置为默认右下角
- 后期：对接数据模型统一管理持久化策略（是否存 `localStorage`、数据结构等届时确定）

## 5. 关键技术决策

| 决策 | 选择 | 替代方案 | 理由 |
|------|------|---------|------|
| 导航模型 | 全屏工具 + 悬浮球 | 仪表盘网格 | 无中间页，切换路径最短；横竖屏天然支持 |
| 工具切换入口 | 卡片堆叠切换器（长按悬浮球） | 底部上滑面板 | iPhone 式交互，3D 层叠契合"漆盒/竹简"意象 |
| 辅助功能入口 | 扇形菜单（短按悬浮球） | 散落各处的按钮 | 一个入口收束所有辅助功能，首屏极简 |
| 返回操作 | 双击悬浮球 | 顶部栏返回按钮 | 放桌上时顶部栏远、悬浮球在右下角拇指热区 |
| 卡片快照 | 静态卡片（名称+图标+主题色） | Keep-Alive 实时快照 / html2canvas | 3D 容器中渲染真实组件有架构障碍；静态卡片零开销 |
| 吸附动画 | `transform: translate()` | `left`/`right` CSS transition | Design System 反模式禁止 `left`/`top` 动画；`left`↔`right` 属性切换有跳变 |
| 动画实现 | Web Animations API（扇形菜单） | CSS transition | 支持随时中断并启动反向动画，CSS transition 中断时需读 computed style |
| 手势事件 | PointerEvents + TouchEvents fallback | 纯 TouchEvents | 桌面 DevTools 调试同时支持鼠标 |
| 拖拽事件绑定 | `pointerdown` 在球上，`move/up` 在 `document` | 全部在球上 | 手指滑出球体后仍能跟踪 |
| 位置持久化 | 会话期内 Pinia 内存存储 | localStorage 持久化 | 本期不做持久化，后期对接数据模型统一管理 |
| 历史栈同步 | Pinia 栈从 Vue Router 派生（`watch route`） | Pinia 独立管理 | 消除浏览器历史与 Pinia 栈不同步的数据一致性问题 |
| 状态管理 | Pinia（按需）+ 组件本地状态 | 全部 Pinia | 简单工具不过度工程化 |
| 路由策略 | `/tool/:name` 动态路由 | 静态 30 条路由 | `tools.map()` 动态生成 |
| 横竖屏 | 都支持 | 仅竖屏 | 悬浮球方案无布局依赖 |
| 暗色模式 | 默认暗色 | 跟随系统 | 面杀主要场景为夜晚室内 |
| 新手引导 | tooltip 气泡 + 扇形菜单前 3 次带标签 | 无引导 | 悬浮球零 affordance，新用户必须引导 |
